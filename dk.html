Философия и назначение

Данный продукт представляет собой законченную экосистему для профессиональных разработчиков, исследователей в области машинного обучения и системных администраторов. Это не просто чат-интерфейс с интерпретатором, а единый рабочий континуум, объединяющий файловый менеджер, профессиональный редактор кода, многопользовательскую песочницу с полной изоляцией, систему контроля версий и распределённую сеть языковых моделей.

Каждый компонент спроектирован с учётом требований к безопасности, скорости отклика и масштабируемости. Приложение работает на базе микрофреймворка Flask, использует расширение Flask-Sock для WebSocket-соединений и обеспечивает живую синхронизацию изменений между пользователями в реальном времени.

Архитектура ядра

Фундамент системы строится на строгой изоляции и контролируемом доступе. Все вычисления выполняются не на хостовой машине, а внутри изолированных контейнеров Docker на базе образа python:3.12-slim. При отключении Docker система автоматически переходит в локальный режим с расширенными проверками безопасности.

  • Многоуровневая аутентификация: Поддержка стандартной авторизации через переменные окружения, OAuth2-интеграция с ВКонтакте и Яндекс.
  • Защита сессий и CSRF: Генерация уникальных идентификаторов и обязательных токенов для всех изменяющих запросов. Полная защита от межсайтовой подделки.
  • Криптография данных: Токены и конфиденциальная информация хранятся в SQLite в зашифрованном виде с использованием симметричного шифрования Fernet. Ключ выводится динамически из секретной строки.
  • Логирование и мониторинг: Ротация дисковых операций для пост-инцидентного анализа. Экспорт метрик в формате Prometheus через эндпоинт /metrics.

Распределённый интеллект

Система использует три специализированные модели от Groq, каждая из которых обладает уникальной ролью, системной инструкцией и стилем генерации. История диалогов привязана к идентификатору модели и сохраняется в базе данных в формате JSON.

Llama 3.3 70B (AI1) — Аналитик. Проводит глубокий структурный анализ кода, выявляет уязвимости, оценивает архитектурные риски и предлагает детальные стратегии рефакторинга.

Mixtral 8x7B (AI2) — Синтезатор. Преобразует разрозненные идеи, заметки и требования в стройную, последовательную проектную документацию и понятные инструкции.

Qwen 2.5 32B (AI3) — Решатель. Генерирует рабочий, практический код с конкретными шагами внедрения, примерами использования и юнит-тестами. Оптимальна для быстрого прототипирования.

Все модели имеют доступ к RAG (Retrieval-Augmented Generation). При включении система сканирует файлы текущего проекта, извлекает релевантные фрагменты и встраивает их в контекст, позволяя ИИ отвечать на основе реальной кодовой базы.

Среда разработки и файловая система

Клиентская часть построена на редакторе Monaco, обеспечивающем подсветку синтаксиса, автодополнение через LSP-подобный протокол и диагностику ошибок в реальном времени. Все изменения фоново автосохраняются каждые тридцать секунд. При повторной загрузке файла система предлагает восстановить предыдущую версию, предотвращая потерю данных.

Файловая система организована вокруг изолированных директорий проектов. Поддерживаются полный перебор каталогов, создание, переименование, перемещение и удаление. Все пути строго проверяются на соответствие корневой директории проекта для предотвращения path traversal-атак.

  • Статический анализ перед запросом: Подсчёт строк, функций, классов и импортов. Выявление опасных конструкций.
  • Diff и Ghost Text: Распознавание паттернов SEARCH/REPLACE в ответах ИИ для точечного применения изменений. Встроенные подсказки при наборе кода.
  • Форматирование: Интеграция с Black для Python и базовое приведение стиля для HTML.

Изолированная песочница и Git

При выполнении кода система сначала пропускает его через детектор опасных команд, использующий регулярные выражения для блокировки рекурсивного удаления, вызовов системы, создания сокетов, форков и шелл-команд. Безопасный код выполняется в контейнере с жёстким таймаутом сто двадцати секунд.

Git-интеграция предоставляет полноценный клиент прямо в интерфейсе. Доступны операции статуса, diff, работа с ветками, коммиты с автоматическим добавлением файлов, граф истории и стеш. Уникальная функция — разрешение конфликтов слияния через ИИ: модель анализирует конфликтные маркеры и предлагает финальный вариант текста.

  • Установка пакетов: Динамическое расширение песочницы через pip.
  • Язык выполнения: Python, PowerShell, Bash. HTML открывается в отдельной вкладке предпросмотра.
  • Безопасность: Таймауты, изоляция окружения, валидация ввода.

Совместная работа и облачная интеграция

WebSocket-сервер организует комнаты на основе пары проект:комната. Участники обмениваются сообщениями и транслируют изменения редактора в реальном времени. Это позволяет организовать парное программирование и коллективное обсуждение без сторонних инструментов.

Модуль Яндекс.Диска предоставляет доступ к облачному хранилищу через OAuth2. Пользователь просматривает папки, загружает и скачивает файлы, создаёт директории. Токены хранятся в зашифрованном виде и обновляются автоматически. Диск используется как бэкап-хранилище или источник данных для песочницы.

  • Аудит безопасности: Запуск pip-audit для поиска уязвимостей в зависимостях и bandit для статического анализа кода. Оба инструмента работают в изоляции с таймаутом.
  • Лимиты использования: Тридцать запросов к чату и двадцать пять к подсказкам в сутки. Сброс каждые двадцать четыре часа.

Инструкции по развёртыванию

Система готова к использованию в production. Развёртывание возможно на локальной машине или корпоративном сервере. Для запуска требуется Python 3.10+, установленные зависимости из requirements.txt, а также настройки переменных окружения в файле .env.

  • Установите библиотеки: pip install -r requirements.txt
  • Настройте .env: укажите GROQ_API_KEYS, AUTH_USER, AUTH_PASS, DATA_ENCRYPTION_KEY.
  • При необходимости включите HTTPS и Docker через соответствующие флаги.
  • Запустите сервер: python main.py. Приложение автоматически откроется в браузере на локальном порту.

Все компоненты взаимодействуют через единую точку входа. Интерфейс адаптивен, поддерживает динамическое переключение темы без перезагрузки. Чат рендерит Markdown-структуры, блоки кода снабжены кнопками копирования и вставки.
